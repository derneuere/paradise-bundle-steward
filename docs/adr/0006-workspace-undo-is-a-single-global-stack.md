# Workspace undo/redo is a single global stack

The pre-**Workspace** editor used per-`(handlerKey, index)` undo stacks — every resource instance had its own private timeline. Multi-**Bundle** could have extended this to per-`(bundleId, handlerKey, index)`, or split into per-**Bundle** stacks; we considered both. We chose instead a single chronological stack covering every edit anywhere in the **Workspace**: ⌘Z undoes the most recent edit, regardless of which **Bundle** or which **Resource** it touched. This matches the universal text-editor / Figma mental model, avoids the original audit's friction observation #6 (per-resource undo is confusing when the user switches focus mid-edit), and lets the `WorkspaceContext` keep one history data structure instead of a tree of them. The cost — losing the ability to "undo only this resource's edits" — is real but no user has asked for it; if it ever surfaces, a per-resource scoped-undo command can be layered on top of the global stack as a view, without touching the storage shape.
