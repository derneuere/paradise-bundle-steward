# WorldViewport overlay selection is the schema `NodePath`

When designing the WorldViewport seam we considered keeping per-resource selection types (`TrafficDataSelection`, `AISectionSelection`, `ZoneSelection`, etc.) and co-locating each overlay's path↔selection translation next to it. We chose instead to make `NodePath` the sole selection currency every WorldViewport overlay receives and emits — overlays match path shapes directly in their click and highlight logic. This unifies "what is the user pointing at?" across the whole schema editor (the same vocabulary the hierarchy tree and inspector already use), makes overlays pure functions of `(data, path) → scene` for testing, and collapses the per-resource translation switch that previously lived in `ViewportPane.tsx`.
